Detailed Description

Set of API calls to operate on individual device registers.

Registers can be access as complete reads and writes, or individual bits can be accessed, while leaving other bits unchanged.

void cs_device_set_wait_repeats ( int  n_wait_repeat_count)

Number of repeat register checks the library will do when waiting on bits to change in a register.

This applies to both the explicit cs_device_wait calls, and implicit library functionality such as ETM programming that requires waiting on bits.

Library default is 32.

Parameters
n_wait_repeat_countnumber of times a register will be checked for bit value

Wait for bit(s) in a device register, to achieve a given state.

Wait function defined by the enum values in cs_reg_waitbits_op_t. Value of register on match (or last failed match value) can be returned if p_last_val pointer is set to a valid location to store the value.

Parameters
devdevice descriptor
offsetregister address offset within the device, in bytes.
bit_maskbit(s) to wait on - bits set to 1 in the mask will be used.
operationtest operations to use. See the CS_REG_WAITBITS_* enum values.
patternbit pattern to wait for in masked bits. Only used if CS_REGWAIT_BITS_PTTRN used as operation.
p_last_valpointer to storage for the last read value of the register.

Write a value to a device register using a bitmask.

Uses a read-modify-write operation. The value written is masked with the bitmask - only the bits set in the mask are changed in the written value.

Parameters
devdevice descriptor
offsetregister address offset, in bytes
datavalue to write to the register.
bitmaskbits to write - a '1' bit in the mask will write the bit from the data value.
